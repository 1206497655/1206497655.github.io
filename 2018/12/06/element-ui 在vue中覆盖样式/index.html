element-ui在vue中覆盖样式
element
前端
发布日期:   2018-12-06
阅读次数:  

vue主张使用scoped关键字来解决css样式的全局污染问题，这样的解决方式明显简单了许多，不过也有一些意外情况，例如想要覆盖一些第三方ui的样式（如elementUi）时，因为大部分的样式都是通过全局样式引入的，在某个页面想进行样式覆盖又存在scoped关键字时显然无法进行覆盖。解决方式有两种：

1.去掉 scoped 提升样式至全局。但是这样的话需要增加命名空间以解决污染问题。

.article-page {
  /* 你的命名空间 */
  .el-tag {
    /* element-ui 元素*/
    margin-right: 0px;
  }
}

2.使用深度选择器。当你子组件使用了 scoped 但在父组件又想修改子组件的样式可以 通过 >>> 来实现：

<style scoped>
.a >>> .b { /* ... */ }
</style>

将会编译成：

.a[data-v-f3f3eg9] .b {
  /* ... */
}

注意：当使用了sass等css预处理语言时， >>> 可能不会生效，此时可以使用 /deep/ 替换，它的作用跟 >>> 时一样的。

除此之外，父级选择器应该是能够被选中的全局样式，如上的 .a ，它应该组件最顶层的父级容器样式，能够被全局样式判定，否则深度选择器是找不到目标样式的。假如组件的样式结构如下：

.article-page {
  /* 你的命名空间 */
  .el-tag {
    /* element-ui 元素*/
    margin-right: 0px;
  }
}

那么深度选择器应该从 .a 样式作为父级选择器开始，比我要改变 .c 样式，从以下方式改变是不行的:

<style scoped language="scss">
    .b /deep/ .c {
        /* 修改失败，无法找到 .c 元素, 父级元素未知 */
    }
</style>

正确的做法应该是这样的:

<style scoped language="scss">
    .a /deep/ .b /deep/ .c {
        /* 修改成功 */
    }
</style>

以上结果基于vue + elementUi测试获得，其它ui未知。欢迎纠错。
